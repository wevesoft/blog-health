# 계산기와 블로그 연동

## 1. 기본 원칙

- **1 성분 = 1 블로그 글 + 1 맞춤 계산기** (가능한 경우)
- 블로그 글의 “권장량” 문단에서 **계산기 링크**를 반드시 넣는다.
- 계산기 페이지에는 **해당 성분 블로그 글 링크**를 넣어 서로 연결한다.

## 2. 블로그 → 계산기

- 글 내 문장 예:  
  “일일 권장량은 성별·운동량에 따라 다릅니다. **→ [내 맞춤 마그네슘 권장량 계산하기](./magnesume_calc.html)**”
- 버튼/카드 형태로 한 번 더 노출 (글 하단 또는 사이드바).

## 3. 계산기 → 블로그

- 계산기 상단 또는 결과 아래에:  
  “이 성분이 궁금하다면 **→ [마그네슘 완벽 가이드](./blog/magnesium.md)**”
- MD 블로그를 웹으로 렌더링하는 경우 경로만 웹 URL로 바꾸면 됨.

## 4. 링크 규칙 (자동화용)

- 계산기 파일: 프로젝트 루트 또는 `calculators/` 에 `{성분영문}_calc.html`  
  예: `magnesume_calc.html`, `vitamin-d_calc.html`
- 블로그 글: `doc/blog/` 또는 `content/blog/` 에 `{성분영문}.md`  
  예: `doc/blog/magnesium.md`
- 상대 경로 예 (doc에서 계산기로): `../magnesume_calc.html` 또는 `../calculators/magnesium_calc.html`

## 5. 성분–계산기 매핑

성분별 계산기 경로는 **ingredient-calc-map.md** 에 정리해 두었습니다.  
새 계산기를 만들면 해당 파일에 한 줄 추가하고, 자동화 스크립트에서 이 매핑을 읽어 링크를 생성하면 됩니다.
