# 자동화 방안

## 1. 콘텐츠 자동화

### 1.1 글 구조 자동 생성

- **입력**: 성분명(한/영), 참고 문헌 URL, 권장량 수치(남/여 등)
- **출력**: `01-blog-strategy.md` 의 템플릿에 맞춘 MD 초안
- **도구**: 스크립트(Python/Node) 또는 AI 프롬프트로 “성분 OOO, 권장량 OOO” 넣고 섹션별 초안 생성 → 사람이 검수·수정

### 1.2 계산기–블로그 링크 일괄 삽입

- **ingredient-calc-map.md** 를 파싱해서:
  - 각 성분 MD 파일에 “맞춤 계산기” 링크 문단이 없으면 **해당 계산기 경로**로 문단 추가
  - 각 계산기 HTML에 “블로그 가이드” 링크가 없으면 **해당 블로그 경로**로 추가
- 빌드 시 또는 커밋 전 훅으로 실행 가능.

### 1.3 새 성분 추가 플로우

1. `02-ingredient-ranking.md` 에서 우선순위 확인  
2. `ingredient-calc-map.md` 에 `성분명, calc_path, blog_path` 추가  
3. 블로그 템플릿으로 MD 초안 생성  
4. 계산기 HTML 복제 후 로직·문구만 해당 성분에 맞게 수정  
5. 링크 일괄 삽입 스크립트 실행  

---

## 2. 배포·빌드 자동화

- **정적 사이트**: MD → HTML 변환 (Hugo, Eleventy, VitePress 등) 후 GitHub Actions로 빌드·배포
- **단일 HTML 계산기**: 그대로 서버에 업로드하거나, 빌드 시 `public/` 또는 `docs/` 로 복사
- **경로 통일**: 블로그 base URL 한 곳에서 관리 (예: `config.baseUrl`) → 계산기·블로그 상호 링크에 사용

---

## 3. 수익·트래픽 측정 자동화

- **GA4 / Search Console**: 페이지별 이벤트로 “계산기 사용”, “제휴 링크 클릭” 추적
- **대시보드**: 성분별 조회수·계산기 사용 횟수·제휴 클릭 수를 시트 또는 간단 대시보드로 주기 집계
- **알림**: 일 매출/클릭이 목표치 넘을 때 슬랙·이메일 알림 (선택)

---

## 4. 추천 구현 순서

1. **ingredient-calc-map.md** 유지 + 블로그/계산기 작성 시 수동으로 링크 넣기  
2. **MD 템플릿**으로 새 성분 글 초안 자동 생성 스크립트  
3. **링크 일괄 삽입** 스크립트 (map 기반)  
4. **CI에서 빌드·배포** 연결  
5. **이벤트·수익 로깅** 및 주간 요약

이 순서로 하면 “자동화”를 단계적으로 넣으면서도 콘텐츠 품질을 유지할 수 있습니다.
