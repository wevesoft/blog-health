# 웹 인포그래픽 — 한눈에 보기 제안

유저가 **블로그를 아주 쉽게** 볼 수 있도록, 정보를 한 화면·한 페이지에 요약하는 방식을 제안합니다.

---

## 1. 한 페이지 요약형 “인포그래픽” 페이지

### 1.1 컨셉

- **한 성분당 1페이지**: 스크롤만 해도 “역할 → 권장량 → 형태 → 주의”가 **블록/카드**로 쭉 보이게 구성
- 긴 글은 “자세히 읽기”로 접어 두거나 별도 상세 페이지로 연결

### 1.2 화면 구성 예 (세로 스크롤)

```
[히어로]
  제목: 마그네슘 (Magnesium)
  한 줄: "수면·근육·에너지 대사에 관여하는 필수 미네랄"

[카드 1] 역할·효능 (아이콘 + 3~5개 불릿)
[카드 2] 일일 권장량 (숫자 크게 + 단위) → [맞춤 계산기] 버튼
[카드 3] 형태 선택 가이드 (글리시네이트 / 시트레이트 / 말레이트 짧은 비교표)
[카드 4] 주의·상호작용 (2~3줄)
[카드 5] 참고 문헌 + 면책
[하단] 전체 글 읽기 / 관련 제품 (제휴)
```

- 모바일: 카드가 세로로 1열.  
- PC: 2열 그리드 가능 (예: 왼쪽 요약, 오른쪽 계산기 임베드 또는 링크).

### 1.3 기술 구현 옵션

- **단일 HTML**: 각 성분별 `magnesium.html` 처럼 한 파일에 카드 섹션 + CSS 그리드/플렉스
- **MD + 테마**: MD frontmatter에 `summary_cards` 배열 넣고, 빌드 시 카드 섹션 자동 생성
- **컴포넌트**: Vue/React로 “성분 인포그래픽” 컴포넌트 하나 만들고, 데이터(JSON/YAML)만 바꿔서 재사용

## 2. 블로그(정보) + 계산기 결합

- **상단**: 인포그래픽 요약 (위 카드들)
- **중간 또는 우측**: 같은 페이지에 **계산기 iframe 또는 컴포넌트** 삽입  
  → 스크롤 없이 “읽기 → 바로 계산” 한 페이지에서 완결
- **하단**: 상세 글(블로그 본문) 접기/펼치기 또는 “자세히 보기” 링크

이렇게 하면 “검색 → 한 페이지에서 요약 + 계산 → 필요 시 상세” 흐름으로 **쉽게** 볼 수 있습니다.

---

## 3. 추가 제안 (알파)

- **성분 비교표**: 2~3개 성분을 한 테이블로 (예: 마그네슘 vs 칼슘 vs 아연 — 역할, 권장량, 상한만)
- **퀵 퀴즈**: “나에게 맞는 형태는?” 2~3문항 선택 → 결과로 추천 형태 + 계산기 링크
- **공유용 이미지**: 각 성분 요약을 **OG 이미지**로 자동 생성 (카드 레이아웃 그대로 이미지화) → SNS·메신저 공유 시 한눈에 보이게

이 방향으로 만들면 “블로그(정보) + 계산기”를 한눈에 들어오는 **인포그래픽형 웹**으로 차별화할 수 있습니다.
